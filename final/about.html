C#alaga

The game's name is a play on the Japanese arcade game, Galaga(play).

The game was coded in Microsoft Visual Studio, C# using Console functions only. Graphics are entirely ANSI.

This app was created for the MATC ITDEV110 course final project. As this is the only opportunity in the Software Development course I have to develop in C#, my goal was to push what I had learned as far as I could.

The goal was to simulate a typical 'game' feel in the console similar to DOS games that I played in High School. From what we had learned in class, player movement would be fairly simple. Getting the app to function while waiting for the player's Console.KeyPress(), however was the initial hurdle.

The solution was found outside of class in the Task Class. Tasks are a way of performing asynchronous operations. This allowed the app to both wait for user input while continuing to function.

Additionally, the use of timers utilizing Unix Epoch in milliseconds were used. These allowed just about every action in the game to be throttled by having a related timer. By doing this, I could have complete control over the flow of every element of the game and could adjust it as needed during development or variably within the game during operation.

Barring the above conventions, everything else that is in this code is what was learned in class. Movement is displayed using Console.WriteLine() and Console.SetCursorPosition(x, y). MVC methodology is utilized to setup methods in a logical fashion.

There is no progression besides Enemy actors being spawned and the eventual player death. A score is calculated by adding one to a counter value every time a player projectile collides with an Enemy actor. Once the Player's health reaches 0 the game gives all actor values a 0 to deactivate them, resets all other variables to their original value and returns to the main menu.

Actors

Player

Player unit. This is the only actor that doesn't have an array associated with the unit itself. The player unit moves whenever the arrow keys are pressed in the direction dictated.

The directional keys dictate movement, 'Q' fires projectiles and 'Esc' opens the pause menu. The player starts with 100 health.


Enemy

The Enemies in the game are comprised of many parallel arrays. Their parent array dictates if they're active(1) or not(0) a value greater than 10 means they are a Power-Up actor.
Enemies have no health. They deactivate immediatly on collision with either the Player, another Enemy or a Player Projectile.

Their spawn rate fixed at .5 seconds, however this is coded to be easily altered during the game if desired. Movement speed, direction and shooting frequency is random to give the game a more organic feel.

The difficulty setting will give the Enemy projectiles an increased X directional bias toward the Player when the projectile was first shot.

Power-Ups

Health

Health Power-Up. If the Player collides with this actor, the Player's Health total will increase by 10 capping at 100.


Explosive Shot

Explosive Shot Power-Up. If the Player collides with this actor, all enemies will spawn Player Projectiles in a radial pattern around the enemy when a Player Projectile collides with and deactivates an Enemy actor.


Spread Shot

Spread Shot Power-Up. If the Player collides with this actor, the Player will spawn two additional Player Projectiles when shooting. If the Player collides with this actor a second time, two more Player Projectiles will spawn with each shot for a total of five.


Once a Player has collected the maximum amount of a Power-Up type, that type will no longer spawn. For example, if the Player has picked up two Spread Shots and one Explosive Shot, only Health will spawn.
